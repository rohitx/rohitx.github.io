Installation Guide to iPython Notebook

January 28, 2014

The installation of iPython notebook on a mac may seem a little more tricky than on Windows or Linux. In other OS, it is either double click or use apt-get install. However, for the mac, a sequeunce of steps need to be followed. Here, I highlight the steps that will help guide you to install iPython notebook on a mac.

I assume that you have the latest version of Python installed. Here are the steps to follow: First install pyzmq. For all of the installations you will need root access. So, install the first package with pip (or easy_install) in the following way: 

sudo pip install pyzmq
Next install Tornado. In my installation of Python, this was already installed. So, if you try to install it, you may get a message saying that it is already installed. Nonetheless, this step ensures that you have a version installed. Tornado is requires to support current version of browsers. 

pip install tornado

Next install jinja2. Ninja is a template tool for rendering HTML pages. 

pip install jinja2

That's it. Once these packages have been installed, you can test iPython Notebook installation with the following command in a terminal: 

ipython notebook

Searching for files like the FILE_SEARCH() function in IDL

January 15, 2014

One easy way to search for files is to make use of the python procedure glob.glob(). This function is very similar to IDL's file_search(). Here's how to use it: 

a = glob.glob("*/*.py")
type(a)
list

Printing out a will give you the list of files that match the search query. 

glob.glob("fullpath pattern")

How to read IDL .sav files in Python

January 04, 2014

With my on-going effort to transition from IDL to Python, I present a post on reading IDL .sav files in Python

  • Make sure you have the latest version of Numpy and Scipy installed
  • Visit this page: IDL Sav file package
  • Open terminal and change directory into your Downloads directory (if this is your default directory
  • Untar the folder calledd "ILDSave-1.0.0.tar.gz"
  • CD into the directory and type: 
    python setup.py install>
    This should install the package.

Next, we use pip to get the IDLsav package. Here's how to do it:

pip install idlsave

Now in the terminal go to the directory where .sav file exists and start python. Once you are inside python do the following: 

import idlsave
s = idlsave.read("filename.sav")

NOte that this case the variable is called "all". This .sav file is a structure. You might get a bunch of errors. Have a look at those and decide if you can ignore them. In most cases you can. Now if you do: 

s.all.type
dtype([(('name', 'NAME'), '|O8'), (('ra', 'RA'), '>f8'), (('de', 'DE'), '>f8'), (('_2mass', '_2MASS'), '|O8'), (('pmra', 'PMRA'), '>f8'), (('pmde', 'PMDE'), '>f8'), (('vmag', 'VMAG'), '>f8'), (('jmag', 'JMAG'), '>f8'), (('hmag', 'HMAG'), '>f8'), (('kmag', 'KMAG'), '>f8'), (('v_j', 'V_J'), '>f8'), (('visits', 'VISITS'), '>f8'), (('field', 'FIELD'), '|O8'), (('origin', 'ORIGIN'), '>i2'), (('proper_motion_src', 'PROPER_MOTION_SRC'), '|O8'), (('priority', 'PRIORITY'), '>i2'), (('type', 'TYPE'), '|O8'), (('ra2mass', 'RA2MASS'), '>f8'), (('de2mass', 'DE2MASS'), '>f8')])

I get the structure or the headers for each of the columns. Now you can parse each column. For example, I can parse just the RA as: 

ra = s.all.ra

That's it. You have now read and parsed IDL .sav file without running IDL.
